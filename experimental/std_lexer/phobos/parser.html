std.d.parser

This module contains a parser for D source code.

Grammar:
The grammar format used in the documentation of this module generally follows the format used by the ANTLR parser generator.
  • Literals are highlighted in green.
  • Rules are links to their definitions.
  • Tokens and rules can be grouped by parenthesis.
  • An asterisk (*) indicates that the previous rule, token, or group can repeat 0 or more times.
  • A question mark (?) indicates that the previous rule, token, or group will be present either 0 or 1 times.
  • A plus sign (+) indicates that the previous rule, token, or group repeats one or more times. (i.e. it is optional)
  • If there is more than one way to match a rule, the alternatives will be separated by a pipe character (|).
  • Rule definitions begin with the rule name followed by a colon (:). Rule definitions end with a semicolon (;).

The grammar for D starts with the module rule.

Examples:
import std.d.lexer;
import std.d.parser;
import std.d.ast;
import std.array;

string sourceCode = q{
import std.stdio;

void main()
{
    writeln("Hello, World.");
}
}c;
void main()
{
    LexerConfig config;
    auto tokens = byToken(cast(ubyte[]) sourceCode, config).array();
    Module mod = parseModule(tokens);
    // Use module here...
}

     | addExpression ('+' | '-' | '~') mulExpression
     ;
